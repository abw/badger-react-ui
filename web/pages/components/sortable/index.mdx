## Typescript Notes

The above examples are written in Typescript.  If you're using plain old
Javascript then you can ignore the types.

The elements in the `items` array that you pass to the components should
extend the `SortableDataItem` type.  This asserts that each item has an `id`
property (number or string) to uniquely identify it.  Your `setOrder` function
should receive an array of these typed items as the argument (e.g.
`items: YourType[]`).

The `Item` component you define should type the properties using the
`SortableItemProps` generic type, specifying your item data type as a type
parameter (e.g. `SortableItemProps<YourType>`).

The `Sortable`, `VerticalSort` and `HorizontalSort` components also accept
a type parameter (e.g. `<Sortable<YourType> ... >`) although it's not usually
required as Typescript can infer the type from the `item`, `Item`
and `setOrder` properties.
